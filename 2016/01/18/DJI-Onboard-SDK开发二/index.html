DJI-Onboard-SDK开发二

发表于 2016-01-18   |   分类于 UAV   |  

本文主要描述运行官方demo中的ROS版本 的基础功能。

开发环境

硬件

  • DJI Matrice 100
  • DJI Manifold
  • ZENMUSE X3

软件

  • Ubuntu 14.04 LTS
  • ROS indigo

操作步骤

安装ros系统

安装步骤

Manifold预装系统是ubuntu14.04并且是arm架构，建议安装ros-armhf-indigo版本,可以根据提示一步一步的进行安装，这里就不赘述了。

配置操作

注意在indigo之后版本的ros系统的精简版中rosbash包（包括roscd、rosrun等命令）是不预置，需要自己搜索安装。
先使用apt-cache search rosbash来找到对应的安装包，需找到安装命令利用sudo apt-get install ros-indigo-rosbash 安装即可。
之后再运行source /opt/ros/indigo/setup.bash，或者如果已经加入此条命令在.bashrc中，则source ~/.bashrc即可。

运行ros例程

创建catkin工作区

如果刚刚安装了catkin需要先运行一下一下命令

$ source /opt/ros/indigo/setup.bash

创建文件夹并初始化

$ mkdir -p ~/catkin_ws/src
$ cd ~/catkin_ws/src
$ catkin_init_workspace

进入到了catkin_ws目录下编译，以后所有编译操作都在此目录。

$ cd ~/catkin_ws/
$ catkin_make

然后查看环境变量，看看工作区路径是否已经存在于环境变量

$ echo $ROS_PACKAGE_PATH
/home/ubuntu/catkin_ws/src:/opt/ros/indigo/share:/opt/ros/indigo/stacks

git获取例程并编译

git下载再拷贝并编译，每次运行需要source一次devel中的setup.bash,使得系统识别工作区的包。

$ cd ~/Downloads
$ git clone https://github.com/dji-sdk/Onboard-SDK.git
$ cp -R Onboard_SDK_Sample/DJI_Onboard_API_ROS_Sample
~/catkin_ws/src/dji_ros
$ cd ~/catkin_ws
$ catkin_make
$ source devel/setup.bash

接着查看包是否被加载

$ rospack find dji_sdk

如果能够查看到路径，说明已经demo包已经被系统识别。

修改源码

进入launch文件修改配置文件

rosed dji_sdk/launch/sdk_manifold.launch.

修改方案如下

  • serial_name 为串口设备名， Manifold 的串口名默认为 /dev/ttyTHS1
  • baud_rate 改成 使能 SDK 中配置的波特率，默认为 230400
  • app_id, enc_key, app_api_level 分别填入登记 APP 获得的 APP
    ID， Communication Key 及对应 level
  • app_version 及 app_bundle_id 在目前的版本中并未使用，保持默认即
    可

运行例程

输入命令启动核心节点

$ roslaunch dji_sdk sdk_manifold.launch

然后再运行用户交互节点

$ rosrun dji_ros dji_ros_client

之后的操作和Linux例程的操作类似。

注意如果没有连接机身的情况下会出现 NO RESPONSE: 374
重新连接即可。还有注意linux例程提到所需要满足的三个操作，否则也会出现类似问题。
